# Trilhas Operacionais

As trilhas operacionais traduzem a inspiracao de management games para gestao
real. Elas nao sao conquistas decorativas. Cada trilha precisa gerar uma
recompensa operacional concreta.

## Estrutura de uma trilha

Cada trilha deve ter:

- missao pequena;
- esforco estimado;
- progresso;
- recompensa real;
- capacidade desbloqueada;
- evidencia do motivo;
- botao para agir.

Exemplo:

```text
Missao: Cotacao piloto
Esforco: baixo
Recompensa: primeiro ciclo real de compra assistida
Desbloqueia: historico de cotacoes, comparacao sugerido vs cotado e aprendizado de embalagem
Evidencia: 72 fornecedores prontos e 261 itens cotaveis
```

## Primeiras trilhas

- Cotacao piloto.
- Mapa fornecedor x marcas.
- Pedido minimo do fornecedor piloto.
- Limpar foco do mix ativo.
- Referencia para cotacao.

## Principio de UX

O usuario deve sentir que esta evoluindo a empresa, nao preenchendo cadastro.

O app deve evitar listas gigantes de pendencias e preferir:

```text
faca esta pequena missao -> veja este ganho -> desbloqueie esta nova inteligencia
```

## Recompensas reais

As recompensas devem falar a linguagem do varejo:

- menos ruptura;
- menos compra quebrada;
- menos tempo perdido;
- cotacao mais clara;
- estoque mais limpo;
- fornecedor mais confiavel;
- capital melhor alocado;
- cliente recuperado;
- previsao mais confiavel.
